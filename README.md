ClearCommute is a modern web application that bridges standard map navigation with real-time environmental data. Instead of just finding the fastest route, ClearCommute calculates the *healthiest* route by sampling PM2.5 air quality indices (AQI) along various paths, ensuring users can minimize their exposure to harmful pollutants during their daily travels.

## ✨ Key Features

- **Pollution-Aware Routing:** Generates multiple routes via OSRM and samples PM2.5 data at intervals along the path to calculate average exposure.
- **Smart Forecasting Tips:** Analyzes 12-hour hourly forecast data and alerts users if leaving slightly later will significantly drop their pollution exposure.
- **Interactive Glassmorphic UI:** A sleek, responsive dark-mode interface built with React and Tailwind-inspired custom CSS.
- **Saved Routes (Authenticated):** Secure user authentication via Supabase allowing users to save and easily reload their daily commutes.
- **Geocoding & Location Services:** Real-time location fetching and reverse geocoding for intuitive start/end point selection.

## 🛠️ Tech Stack

- **Frontend Framework:** Next.js / React
- **Mapping & Rendering:** Leaflet (via `react-leaflet`)
- **Routing Engine:** OSRM (Open Source Routing Machine) API
- **Air Quality & Weather:** Open-Meteo API
- **Authentication & Database:** Supabase
- **Deployment:** Vercel

## 🚀 Getting Started

Follow these steps to set up the project locally.

### Prerequisites
- [Node.js](https://nodejs.org/en/) (v18 or higher recommended)
- [Git](https://git-scm.com/)
- A [Supabase](https://supabase.com/) account (for Auth and Database)

### Installation

1. Clone the repository
   
   git clone [https://github.com/yourusername/clear-commute-app.git](https://github.com/yourusername/clear-commute-app.git)
   cd clear-commute-app

2. Install dependencies

   npm install


3. Configure Environment Variables
   Create a .env.local file in the root directory and add your Supabase credentials:
   NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key

4. Run the development server

   npm run dev
   Open http://localhost:3000 in your browser to see the application.


### Database Schema

To enable the "Saved Routes" feature, execute the following SQL in your Supabase SQL Editor:

create table saved_routes (
  id bigint generated by default as identity primary key,
  user_id uuid references auth.users not null,
  name text not null,
  start_point jsonb not null,
  end_point jsonb not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

alter table saved_routes enable row level security;
create policy "Users can view their own routes" on saved_routes for select using (auth.uid() = user_id);
create policy "Users can insert their own routes" on saved_routes for insert with check (auth.uid() = user_id);
create policy "Users can delete their own routes" on saved_routes for delete using (auth.uid() = user_id);


###DEPLOYMENT:

This project is optimized for deployment on Vercel.
Push your code to GitHub.
Import the repository in Vercel.
Add the NEXT_PUBLIC_SUPABASE_URL and NEXT_PUBLIC_SUPABASE_ANON_KEY to Vercel's Environment Variables.
Deploy!

Note: Ensure your Supabase Authentication settings have your production Vercel URL added to the Site URL and Redirect URLs to allow secure logins.
